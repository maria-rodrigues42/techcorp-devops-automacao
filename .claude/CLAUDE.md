# Modelo & effort (vale para TODOS os projetos deste computador)

No início de cada projeto/tarefa e em cada virada de fase (arquitetura → implementação mecânica → revisão/debug), **recomende proativamente o melhor modelo e o melhor effort**, com uma linha de justificativa.

**Importante: não consigo trocar o modelo nem o effort sozinho** — quem troca é a usuária via `/model`, `/effort` (e `/fast`). Portanto, ao chegar num ponto de troca que eu recomendei, **avise explicitamente** (ex.: "agora troque para Sonnet 4.6 + medium") para ela rodar o comando.

Heurística padrão:
- **Sonnet 4.6 + medium** — trabalho bem-especificado/mecânico (aplicar um plano já escrito, refactor repetitivo).
- **Opus 4.8 + high** — arquitetura, design ambíguo, ou depurar bug difícil.
- **max** — só quando realmente travou (queima cota rápido).
- **Haiku 4.5 + low/medium** — consultas rápidas/baratas, status, comandos simples.
